# City Bus Management System - Product Requirements Document

## 1. Project Overview
The City Bus Management System is a web-based application designed to manage the daily operations of a city's public transit network. It allows administrators to manage bus routes, vehicles, drivers, and schedules efficiently.

## 2. User Roles
- **Administrator**: Has full access to all modules (Routes, Buses, Drivers, Schedules, Users).

## 3. Core Features

### 3.1 Dashboard
- Overview of total routes, active buses, and on-duty drivers.
- Quick status summary.

### 3.2 Route Management
- **List Routes**: View all bus routes with details (Route Number, Start/End Stations, Operating Hours).
- **Add/Edit Route**: Create new routes or modify existing ones.
- **Delete Route**: Remove obsolete routes.
- **Stop Management**: Manage stops associated with each route.

### 3.3 Bus Management
- **Vehicle List**: Registry of all buses (License Plate, Capacity, Model, Status).
- **Maintenance Status**: Track if a bus is Active, In Maintenance, or Retired.
- **Add/Edit Bus**: Update vehicle information.

### 3.4 Driver Management
- **Driver List**: Information about drivers (Name, License Number, Contact, Status).
- **Status Tracking**: On-duty, Off-duty, On Leave.
- **Add/Edit Driver**: Manage driver personnel records.

### 3.5 Scheduling (Dispatch)
- **Assign**: Assign drivers and buses to specific routes for specific time slots.
- **View Schedule**: Calendar or list view of daily operations.

## 4. Non-Functional Requirements
- **Performance**: Fast loading times for data tables.
- **Usability**: Clean, intuitive interface for administrators.
- **Responsive**: Accessible on desktop and tablet devices.
